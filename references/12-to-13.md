## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "laravel/framework": "^13.0"
    },
    "require-dev": {
        "phpunit/phpunit": "^12.0",
        "pestphp/pest": "^4.0"
    }
}
```

Then run:
```bash
composer update
```

Update the global Laravel installer:
```bash
composer global update laravel/installer
```

**PHP minimum version:** 8.2.0 (unchanged from v12)

## 2. High Impact Changes

### 2a. CSRF Middleware Renamed to `PreventRequestForgery`

The `VerifyCsrfToken` middleware has been renamed to `PreventRequestForgery`. It also now adds **request-origin verification** using the `Sec-Fetch-Site` header.

`VerifyCsrfToken` and `ValidateCsrfToken` remain as **deprecated aliases** — they still work but should be updated.

Update any test exclusions or route definitions that reference the old class:

```php
use Illuminate\Foundation\Http\Middleware\PreventRequestForgery;

// Before
->withoutMiddleware([\App\Http\Middleware\VerifyCsrfToken::class]);

// After
->withoutMiddleware([PreventRequestForgery::class]);
```

The middleware configuration API also adds `preventRequestForgery(...)`.

## 3. Medium Impact Changes

### 3a. Cache `serializable_classes` Security Hardening

A new `serializable_classes` option has been added to `config/cache.php`. It defaults to `false`, which **blocks unserialization of PHP objects from cache** unless they are explicitly allowed. This prevents potential PHP deserialization gadget chain attacks if your `APP_KEY` is ever leaked.

**If your app stores PHP objects in cache**, you must explicitly allow them:

```php
// config/cache.php
'serializable_classes' => [
    App\Data\CachedDashboardStats::class,
    App\Support\CachedPricingSnapshot::class,
],
```

If nothing is cached as PHP objects (arrays, scalars, JSON-decoded data), no action is needed.

To allow *all* objects (not recommended):
```php
'serializable_classes' => true,
```

## 4. Low Impact Changes

### Cache: Key Prefix and Session Cookie Name Format Changed

The auto-generated default cache prefix and session cookie name formats have changed. This **only affects apps that rely on framework-generated defaults** (i.e., no `CACHE_PREFIX`, `REDIS_PREFIX`, or `SESSION_COOKIE` set in `.env`).

```php
// Laravel <= 12.x (used underscores)
// cache prefix:   "my_app_cache_"
// session cookie: "my_app_session"

// Laravel >= 13.x (uses hyphens for prefix, snake_case for session)
// cache prefix:   "my-app-cache-"
// session cookie: "my_app_session"  (snake_case via Str::snake)
```

To lock in your current values and avoid any change, explicitly set these in `.env`:
```
CACHE_PREFIX=my_app_cache_
SESSION_COOKIE=my_app_session
```

### Container: `Container::call` Respects Nullable Defaults

`Container::call` now respects nullable class parameter defaults (matching the v12 constructor injection change):

```php
$container->call(function (?Carbon $date = null) {
    return $date;
});

// <= 12.x: Carbon instance (resolved ignoring null default)
// >= 13.x: null (default respected)
```

### Database: MySQL `DELETE` With `JOIN` / `ORDER BY` / `LIMIT`

Laravel now correctly compiles `DELETE ... JOIN` queries including `ORDER BY` and `LIMIT` for MySQL. Previously these clauses were silently ignored on joined deletes.

If you use `JOIN` + `DELETE` + `ORDER BY`/`LIMIT` together, verify your queries still behave as expected. Some MySQL/MariaDB variants may throw if the syntax is unsupported.

### Eloquent: Polymorphic Pivot Table Name Pluralization

When pivot table names are inferred for polymorphic `MorphToMany` relationships using custom pivot model classes, Laravel now generates **pluralized** table names.

If you relied on the old singular inferred names, explicitly define `$table` on your pivot model:

```php
class RoleUser extends MorphPivot
{
    protected $table = 'role_user'; // lock in the old singular name
}
```

### Eloquent: Collection Serialization Restores Eager-Loaded Relations

When Eloquent model collections are serialized (e.g., in queued jobs) and then restored, eager-loaded relations are now preserved on the restored models.

If your code assumed relations would be absent after deserialization, adjust accordingly.

### Queue: `JobAttempted` Event — `$exceptionOccurred` → `$exception`

```php
// Before
$event->exceptionOccurred; // bool

// After
$event->exception; // Throwable|null
```

Update any listeners on `Illuminate\Queue\Events\JobAttempted`.

### Queue: `QueueBusy` Event — `$connection` → `$connectionName`

```php
// Before
$event->connection;

// After
$event->connectionName;
```

Update any listeners on `Illuminate\Queue\Events\QueueBusy`.

### Routing: Domain Routes Take Precedence

Routes registered with an explicit domain are now matched before non-domain routes, regardless of registration order. Review any routing logic that mixed domain and non-domain routes if you relied on a specific precedence.

### Support: `Str` Factories Reset Between Tests

Custom `Str` factories (UUID, ULID, random string) are now reset during test teardown. If tests depended on factories persisting between test methods, move the factory setup into each test or a `setUp` hook.

### Views: Bootstrap Pagination View Names Changed

Internal Bootstrap 3 pagination view names are now explicit:

```php
// Before
'pagination::default'
'pagination::simple-default'

// After
'pagination::bootstrap-3'
'pagination::simple-bootstrap-3'
```

Update any code that references the old view names directly.

## 5. Very Low Impact Changes

### Authentication

- **`MustVerifyEmail` contract** — new `markEmailAsUnverified()` method required on custom implementations
- **`AuthenticationException::redirectTo()`** — already updated in v11; no new changes
- **Default password reset email subject** changed from `"Reset Password Notification"` to `"Reset your password"` — update test assertions or translation overrides if needed
- **Queued notifications** now respect `#[DeleteWhenMissingModels]` / `$deleteWhenMissingModels`

### Cache

- **`Store` and `Repository` contracts** — new `touch($key, $seconds)` method; update custom store implementations

### Contracts

- **`Dispatcher` contract** — new `dispatchAfterResponse($command, $handler = null)` method
- **`ResponseFactory` contract** — new `eventStream(...)` method
- **`Queue` contract** — new size inspection methods: `pendingSize`, `delayedSize`, `reservedSize`, `creationTimeOfOldestPendingJob`

### Eloquent

- **Model booting** — instantiating a model from within its own `boot()` or trait `boot*()` method now throws a `LogicException`. Move such logic outside the boot cycle.

### HTTP Client

- `Response::throw()` and `throwIf()` now declare callback parameter types in their signatures — ensure custom response class overrides are compatible

### Scheduling

- `ApplicationBuilder::withScheduling()` now defers schedule registration until `Schedule` is resolved — adjust if you depended on immediate registration timing

### Support

- **Manager `extend` callbacks** are now bound to the manager instance (`$this`). If you used `$this` inside an `extend` closure expecting a different bound object, use `use (...)` captures instead
- **`Js::from`** now uses `JSON_UNESCAPED_UNICODE` by default — update test assertions that expected escaped Unicode sequences (e.g. `\u00e8`)
