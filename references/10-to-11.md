## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "laravel/framework": "^11.0",
        "laravel/sanctum": "^4.0"
    },
    "require-dev": {
        "nunomaduro/collision": "^8.1"
    }
}
```

Also install optional packages if used:

```json
"laravel/breeze": "^2.0",
"laravel/cashier": "^15.0",
"laravel/dusk": "^8.0",
"laravel/jetstream": "^5.0",
"laravel/octane": "^2.3",
"laravel/passport": "^12.0",
"laravel/scout": "^10.0",
"laravel/telescope": "^5.0",
"livewire/livewire": "^3.4",
"inertiajs/inertia-laravel": "^1.0"
```

Remove `doctrine/dbal` if present — Laravel 11 no longer requires it:
```bash
composer remove doctrine/dbal
```

Update the global installer:
```bash
composer global require laravel/installer:^5.6
```

Then run:
```bash
composer update
```

**PHP minimum version:** 8.2.0
**curl minimum version:** 7.34.0

## 2. High Impact Changes

### 2a. PHP 8.2.0 Required

Ensure your server runs PHP ≥ 8.2.0.

```bash
php --version
```

### 2b. Application Structure

Laravel 11 introduces a new slimmer app skeleton (fewer service providers, middleware, config files). **You do not need to migrate your existing Laravel 10 app structure** — Laravel 11 fully supports the Laravel 10 layout. Only new projects use the new structure by default.

### 2c. Floating-Point Column Types Rewritten

The `double` and `float` migration methods have changed signatures.

**`double` — remove `$total` and `$places` arguments:**

Before:
```php
$table->double('amount', 15, 2);
```

After:
```php
$table->double('amount');
```

**`float` — remove `$total`/`$places`, optionally add `$precision`:**

Before:
```php
$table->float('amount', 8, 2);
```

After:
```php
$table->float('amount');              // 8-byte double-precision (default)
$table->float('amount', precision: 24); // 4-byte single-precision
$table->float('amount', precision: 53); // 8-byte double-precision
```

**Removed methods** — `unsignedDecimal`, `unsignedDouble`, `unsignedFloat` are gone. Chain `->unsigned()` instead:
```php
$table->decimal('amount', total: 8, places: 2)->unsigned();
$table->double('amount')->unsigned();
$table->float('amount', precision: 53)->unsigned();
```

### 2d. Modifying Columns Requires All Attributes

When using `->change()` to modify a column, you must now **explicitly re-declare all attributes** you want to keep. Any attribute not listed in the migration will be dropped.

Before (Laravel 10 — silently preserved existing attributes):
```php
$table->integer('votes')->nullable()->change();
// 'unsigned', 'default', 'comment' were silently preserved
```

After (Laravel 11 — must be explicit):
```php
$table->integer('votes')
    ->unsigned()
    ->default(1)
    ->comment('The vote count')
    ->nullable()
    ->change();
```

> ⚠️ This is one of the most common silent breakages when upgrading. Audit all your `->change()` calls.

**Shortcut** — squash existing migrations to avoid editing them all:
```bash
php artisan schema:dump
```
Laravel will then use the schema dump as the baseline and only run new migrations.

**Index modifiers with `->change()`:**
```php
$table->bigIncrements('id')->primary()->change();    // add index
$table->char('postal_code', 10)->unique(false)->change(); // drop index
```

### 2e. SQLite 3.26.0+ Required

If using SQLite, ensure version ≥ 3.26.0.

```bash
sqlite3 --version
```

### 2f. Sanctum 4.x — Publish Migrations

Sanctum (and other packages) **no longer auto-load their migrations**. You must publish them:

```bash
php artisan vendor:publish --tag=sanctum-migrations
php artisan vendor:publish --tag=cashier-migrations      # if using Cashier
php artisan vendor:publish --tag=passport-migrations     # if using Passport
php artisan vendor:publish --tag=telescope-migrations    # if using Telescope
php artisan vendor:publish --tag=spark-migrations        # if using Spark
```

Then update `config/sanctum.php` middleware references:
```php
'middleware' => [
    'authenticate_session' => Laravel\Sanctum\Http\Middleware\AuthenticateSession::class,
    'encrypt_cookies'      => Illuminate\Cookie\Middleware\EncryptCookies::class,
    'validate_csrf_token'  => Illuminate\Foundation\Http\Middleware\ValidateCsrfToken::class,
],
```

## 3. Medium Impact Changes

### 3a. Carbon 3

Laravel 11 supports both Carbon 2 and Carbon 3. If you upgrade to Carbon 3, be aware of these breaking changes:

- **`diffIn*` methods now return `float` instead of `int`** and can return negative values to indicate direction:
```php
// Carbon 2: always positive int
$diff = $date->diffInDays($other); // e.g. 5

// Carbon 3: float, possibly negative
$diff = $date->diffInDays($other); // e.g. -5.0 or 5.0
```

Review all date diff comparisons in your codebase. Use `abs()` if you relied on positive-only results, or use `->diffInDays($other, absolute: true)`.

See Carbon's [3.0 changelog](https://github.com/briannesbitt/Carbon/releases/tag/3.0.0) for the full list of changes.

### 3b. Password Rehashing on Login

Laravel 11 automatically rehashes user passwords on login when the hashing work factor has changed. This is safe for most apps, but if your `User` model uses a non-standard password field name, declare it:

```php
// In your User model
protected $authPasswordName = 'custom_password_field';
```

To disable rehashing entirely:
```php
// config/hashing.php
'rehash_on_login' => false,
```

### 3c. Per-Second Rate Limiting

Rate limiting now supports per-second granularity. If you manually instantiate rate limiter classes (not common), their constructors now accept **seconds** instead of minutes:

```php
// Before (minutes)
new Limit($key, $attempts, 2);           // 2 minutes
new ThrottlesExceptions($attempts, 2);   // 2 minutes

// After (seconds)
new Limit($key, $attempts, 2 * 60);      // 2 minutes = 120 seconds
new ThrottlesExceptions($attempts, 2 * 60);
```

Also: `Limit::$decayMinutes` property renamed to `Limit::$decaySeconds`.

Documented static constructors (`Limit::perMinute()`, `Limit::perSecond()`) are **unaffected**.

### 3d. `spatie/once` Replaced by Built-in `once()`

Laravel 11 ships a native `once()` helper. Remove the Spatie package to avoid conflicts:

```bash
composer remove spatie/once
```

The built-in `once()` works identically — no code changes needed.

## 4. Low Impact Changes

### Authentication

- **`AuthenticationException::redirectTo()`** now requires an `Illuminate\Http\Request` as its first argument:
```php
// Before
$path = $e->redirectTo();

// After
$path = $e->redirectTo($request);
```

- **`UserProvider` contract** — add `rehashPasswordIfRequired(Authenticatable $user, array $credentials, bool $force = false)` to custom implementations
- **`Authenticatable` contract** — add `getAuthPasswordName()` to custom implementations (return `'password'` by default)
- **`SendEmailVerificationNotification`** is now auto-registered. To prevent auto-registration, add an empty override in `EventServiceProvider`:
```php
protected function configureEmailVerification(): void {}
```

### Cache

- **Cache key prefix** — the automatic `:` suffix after prefixes has been removed for DynamoDB, Memcached, and Redis. If you rely on the old `:` separator, add it manually to your prefix value in config.

### Collections

- **`Enumerable::dump()`** now accepts variadic `...$args` — update custom `Enumerable` implementations

### Database

- **Doctrine DBAL fully removed** — remove any calls to the Doctrine-based methods listed below and switch to native schema methods:

| Removed | Replacement |
|---------|-------------|
| `Schema::getAllTables()` | `Schema::getTables()` |
| `Schema::getAllViews()` | `Schema::getViews()` |
| `Schema::getAllTypes()` | `Schema::getTypes()` |
| `Connection::getDoctrineConnection()` | — |
| `Connection::registerDoctrineType()` | — |
| `DB::registerDoctrineType()` | — |

- **Spatial column types** — old specific types removed; use `geometry` or `geography`:
```php
// Before
$table->point('location');
$table->polygon('area');
$table->multiPolygon('regions');

// After
$table->geometry('location', subtype: 'point');
$table->geometry('area', subtype: 'polygon');
$table->geometry('regions', subtype: 'multipolygon');

// Or for geography (uses SRID 4326 by default on supported DBs)
$table->geography('coordinates', subtype: 'point', srid: 4326);
```

- **Eloquent `casts` method** — the base `Model` class now defines a `casts()` method. Rename any `casts` *relationship* on your models to avoid conflicts.
- **MariaDB dedicated driver** — optionally switch `'driver' => 'mariadb'` in `config/database.php` for MariaDB connections. Note: with the `mariadb` driver, `$table->uuid()` creates a native UUID column (not `char(36)`). If you switch drivers on an existing app, change existing `uuid()` migration calls to `char('column', 36)`.
- **`ConnectionInterface`** — new `scalar()` method required if implementing this interface
- **`Schema::getColumnType()`** now returns the actual DB type, not the Doctrine equivalent

### Mail

- **`Mailer` contract** — add `sendNow($mailable, array $data = [], $callback = null)` to custom implementations

### Packages

- **Publishing service providers** — packages should use `ServiceProvider::addProviderToBootstrapFile(Provider::class)` instead of modifying `config/app.php`, since the `providers` array no longer exists in new Laravel 11 app skeletons

### Queues

- **`BatchRepository` interface** — add `rollBack()` method to custom implementations
- **Synchronous jobs now respect `after_commit`** configuration — previously `sync` driver jobs ran immediately regardless
