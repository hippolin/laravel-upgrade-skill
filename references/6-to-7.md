## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "laravel/framework": "^7.0",
        "laravel/tinker": "^2.0"
    },
    "require-dev": {
        "facade/ignition": "^2.0",
        "nunomaduro/collision": "^4.1",
        "phpunit/phpunit": "^8.5"
    }
}
```

Then run:
```bash
composer update
```

**PHP minimum version:** 7.2.5

## 2. High Impact Changes

### 2a. Symfony 5 — Exception Handler

Update `App\Exceptions\Handler` to accept `Throwable` instead of `Exception`:

**Before:**
```php
use Exception;

public function report(Exception $exception);
public function shouldReport(Exception $exception);
public function render($request, Exception $exception);
public function renderForConsole($output, Exception $exception);
```

**After:**
```php
use Throwable;

public function report(Throwable $exception);
public function shouldReport(Throwable $exception);
public function render($request, Throwable $exception);
public function renderForConsole($output, Throwable $exception);
```

Also update any queued job `failed()` methods:

**Before:**
```php
public function failed(Exception $exception) { ... }
```

**After:**
```php
public function failed(Throwable $exception) { ... }
```

### 2b. Symfony 5 — Session Config

In `config/session.php`, update the `secure` key:

**Before:**
```php
'secure' => env('SESSION_SECURE_COOKIE', false),
```

**After:**
```php
'secure' => env('SESSION_SECURE_COOKIE', null),
```

### 2c. Symfony 5 — Artisan Commands Must Return Integer

**Before:**
```php
public function handle()
{
    return true;
}
```

**After:**
```php
public function handle()
{
    return 0; // 0 = success, non-zero = failure
}
```

### 2d. Authentication Scaffolding

`laravel/ui` must now be in `require` (not `require-dev`) if you use auth scaffolding:

```bash
composer require laravel/ui "^2.0"
```

### 2e. Date Serialization

Eloquent models now serialize dates in ISO-8601 format (UTC with fractional seconds).

**Old format:** `2019-12-02 20:01:00`  
**New format:** `2019-12-02T20:01:00.283041Z`

To keep the old format, override `serializeDate()` in your model(s):

```php
use DateTimeInterface;

protected function serializeDate(DateTimeInterface $date)
{
    return $date->format('Y-m-d H:i:s');
}
```

> ⚠️ This affects `toArray()` and `toJson()`. Database storage is unaffected.
> If your API consumers or frontend depend on the date format, this is a **breaking change**.

## 3. Medium Impact Changes

### 3a. Blade Components — Blade X Conflict

Laravel 7 adds first-class Blade tag component support. If you use the `"blade-ui-kit/blade-icons"` or similar "Blade X" packages and experience conflicts, disable built-in tag components in `AppServiceProvider`:

```php
use Illuminate\Support\Facades\Blade;

public function boot()
{
    Blade::withoutComponentTags();
}
```

### 3b. `Blade::component` Renamed

**Before:**
```php
Blade::component('components.alert', 'alert');
```

**After:**
```php
Blade::aliasComponent('components.alert', 'alert');
```

### 3c. CORS Support

Laravel 7 includes CORS out of the box. Remove any third-party CORS packages and:

1. Install the underlying library:
```bash
composer require fruitcake/laravel-cors
```

2. Add to global middleware in `App\Http\Kernel`:
```php
protected $middleware = [
    \Fruitcake\Cors\HandleCors::class,
    // ...
];
```

3. Publish the config:
```bash
php artisan vendor:publish --tag="cors"
```

Config file: `config/cors.php` — [view example](https://github.com/laravel/laravel/blob/master/config/cors.php)

### 3d. Factory Types Removed

"Factory types" (undocumented since 2016) have been removed. Migrate to [factory states](https://laravel.com/docs/7.x/database-testing#factory-states):

**Before (factory type):**
```php
$factory->defineAs(App\User::class, 'admin', function ($faker) {
    return ['role' => 'admin'];
});
```

**After (factory state):**
```php
$factory->define(App\User::class, function ($faker) {
    return ['role' => 'user'];
});

$factory->state(App\User::class, 'admin', function ($faker) {
    return ['role' => 'admin'];
});
```

Usage:
```php
factory(App\User::class)->states('admin')->create();
```

### 3e. Markdown Mail Templates

Default Markdown mail templates have been redesigned. The undocumented `promotion` component has been removed.

If you've published these templates, re-publish to get the new design:
```bash
php artisan vendor:publish --tag=laravel-mail --force
```

> ⚠️ Markdown mail templates must now use **unindented HTML** (indentation has special meaning in Markdown).

### 3f. `assertSee` Auto-escaping

These test assertion methods now **automatically escape** the value before asserting:
- `assertSee`
- `assertDontSee`
- `assertSeeText`
- `assertDontSeeText`
- `assertSeeInOrder`
- `assertSeeTextInOrder`

**Before (manual escaping):**
```php
$response->assertSee(e('Some <value>'));
```

**After (auto-escaped — remove manual escaping):**
```php
$response->assertSee('Some <value>');
```

To assert **raw/unescaped** values, pass `false` as the second argument:
```php
$response->assertSee('<strong>Bold</strong>', false);
```

### 3g. `different` Validation Rule

The `different` rule now **fails** if one of the specified parameters is missing from the request (previously it would pass silently).

Review usages of `'field' => 'different:other_field'` and ensure both fields are always present when the rule applies.

### 3h. Unique Route Names

Duplicate route names are **no longer allowed** in Laravel 7. Routes sharing the same name will cause unexpected behavior.

Search your routes for duplicates:
```bash
php artisan route:list | sort
```

Fix by assigning unique names:
```php
// Before (duplicate)
Route::get('/users', ...)->name('users');
Route::get('/admin/users', ...)->name('users'); // ❌

// After
Route::get('/users', ...)->name('users.index');
Route::get('/admin/users', ...)->name('admin.users.index'); // ✅
```

## 4. Low Impact Changes

### Eloquent

- **`addHidden` / `addVisible` removed** → use `makeHidden()` / `makeVisible()`
- **`booting` / `booted` model methods added** → rename your methods if they conflict
- **`getOriginal()` now respects casts and mutators** → use `getRawOriginal()` for raw values
- **`resolveRouteBinding($value, $field = null)`** now accepts a `$field` argument — update custom implementations

### Auth

- **`TokenRepositoryInterface`** has a new `recentlyCreatedToken()` method — update custom implementations

### HTTP

- **PSR-7 (Zend Diactoros deprecated)** → switch to `nyholm/psr7`:
```bash
composer require nyholm/psr7
composer require symfony/psr-http-message-bridge "^2.0"
```

### Mail

- **`MAIL_DRIVER` env renamed to `MAIL_MAILER`** — update your `.env` file
- **Swift Mailer bindings** (`swift.mailer`, `swift.transport`) removed from container — access via:
```php
$swiftMailer = app('mailer')->getSwiftMailer();
$swiftTransport = $swiftMailer->getTransport();
```
- New multi-mailer config structure available (backwards compatible, no immediate change required): [view new config](https://github.com/laravel/laravel/blob/7.x/config/mail.php)

### Resources

- `Illuminate\Http\Resources\Json\Resource` removed → extend `JsonResource` instead:
```php
// Before
use Illuminate\Http\Resources\Json\Resource;
class UserResource extends Resource { ... }

// After
use Illuminate\Http\Resources\Json\JsonResource;
class UserResource extends JsonResource { ... }
```

### Routing

- `$router->getRoutes()` now returns `RouteCollectionInterface` instead of `RouteCollection`

### Testing

- `Illuminate\Foundation\Testing\TestResponse` renamed → `Illuminate\Testing\TestResponse`
- `Illuminate\Foundation\Testing\Assert` renamed → `Illuminate\Testing\Assert`

### Session

- `array` session driver now persists data for the current request (previously it didn't)
