## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "laravel/framework": "^9.0",
        "spatie/laravel-ignition": "^1.0"
    },
    "require-dev": {
        "nunomaduro/collision": "^6.1"
    }
}
```

Also replace `facade/ignition` with `spatie/laravel-ignition`:
```bash
composer remove facade/ignition
composer require spatie/laravel-ignition "^1.0" --dev
```

If using Pusher:
```bash
composer require pusher/pusher-php-server "^5.0"
```

Then run:
```bash
composer update
```

**PHP minimum version:** 8.0.2

Project-specific note: this codebase should upgrade PHP to 8.1 before starting the Laravel 9 bump, because the Laravel 9 dependency set and local/container images are aligned to PHP 8.1 here.

## 2. High Impact Changes

### 2a. Flysystem 3.x

Laravel's `Storage` facade now uses Flysystem 3.x (up from 1.x). Key behavioural changes:

**Install driver packages** if you use S3, FTP, or SFTP:
```bash
# Amazon S3
composer require -W league/flysystem-aws-s3-v3 "^3.0"

# FTP
composer require league/flysystem-ftp "^3.0"

# SFTP
composer require league/flysystem-sftp-v3 "^3.0"
```

**Write operations now overwrite by default** — `put`, `write`, `writeStream` will silently overwrite existing files. Check for existence first if needed:
```php
if (!Storage::exists('file.txt')) {
    Storage::put('file.txt', $contents);
}
```

**Write failures return `false` instead of throwing** — if you need exceptions, add `'throw' => true` to your disk config:
```php
// config/filesystems.php
'public' => [
    'driver' => 'local',
    // ...
    'throw' => true,
],
```

**Reading missing files returns `null`** (previously threw `FileNotFoundException`).

**Deleting missing files returns `true`** (previously may have thrown).

**Cached adapters removed** — remove any `cache` key from your disk configurations.

**SFTP passphrase key renamed:**
```php
// Before
'password' => 'my-passphrase',

// After
'passphrase' => 'my-passphrase',
```

**Custom filesystem drivers** must now return `Illuminate\Filesystem\FilesystemAdapter`:

Before (Laravel 8):
```php
Storage::extend('dropbox', function ($app, $config) {
    return new Filesystem(new DropboxAdapter($client));
});
```

After (Laravel 9):
```php
use Illuminate\Filesystem\FilesystemAdapter;

Storage::extend('dropbox', function ($app, $config) {
    $adapter = new DropboxAdapter(new DropboxClient($config['authorization_token']));

    return new FilesystemAdapter(
        new Filesystem($adapter, $config),
        $adapter,
        $config
    );
});
```

### 2b. Symfony Mailer (replaces SwiftMailer)

SwiftMailer is no longer maintained and has been replaced by Symfony Mailer. This is the most involved change in v9.

**Install driver packages** if using Mailgun or Postmark:
```bash
# Mailgun
composer require symfony/mailgun-mailer symfony/http-client

# Postmark (also remove wildbit/swiftmailer-postmark)
composer remove wildbit/swiftmailer-postmark
composer require symfony/postmark-mailer symfony/http-client
```

**Renamed methods** — update any Mailables or listeners that call these:

| Laravel 8 (Swift) | Laravel 9 (Symfony) |
|---|---|
| `withSwiftMessage($cb)` | `withSymfonyMessage($cb)` |
| `Message::getSwiftMessage()` | `Message::getSymfonyMessage()` |
| `Mailer::getSwiftMailer()` | `Mailer::getSymfonyTransport()` |
| `Mailer::setSwiftMailer($s)` | `Mailer::setSymfonyTransport($t)` |
| `MailManager::createTransport()` | `MailManager::createSymfonyTransport()` |

**Message building methods renamed** (if using low-level message manipulation):

Before:
```php
$this->withSwiftMessage(function ($message) {
    $message->getHeaders()->addTextHeader('X-Custom', 'value');
});
```

After:
```php
use Symfony\Component\Mime\Email;

$this->withSymfonyMessage(function (Email $message) {
    $message->getHeaders()->addTextHeader('X-Custom', 'value');
});
```

**Proxied message methods renamed:**

Before:
```php
$message
    ->setFrom('sender@example.com')
    ->setTo('recipient@example.com')
    ->setSubject('Hello')
    ->setBody('<h1>Hi</h1>', 'text/html')
    ->addPart('Plain text', 'text/plain');
```

After:
```php
$message
    ->from('sender@example.com')
    ->to('recipient@example.com')
    ->subject('Hello')
    ->html('<h1>Hi</h1>')
    ->text('Plain text');
```

**SMTP stream config flattened:**

Before (`config/mail.php`):
```php
'stream' => [
    'ssl' => ['verify_peer' => false],
],
```

After:
```php
'verify_peer' => false,
```

**`send` / `html` / `raw` / `plain` return type changed** — these methods on `Illuminate\Mail\Mailer` now return `Illuminate\Mail\SentMessage` instead of `void`.

**`MessageSent` event** — the `$message` property is now a `Symfony\Component\Mime\Email` instance (pre-send). A new `$sent` property (`Illuminate\Mail\SentMessage`) contains post-send info including the message ID.

**`auth_mode` no longer needed** in mail config — Symfony Mailer negotiates it automatically.

**Failed recipients** — no longer retrievable after sending. A `TransportExceptionInterface` is thrown instead. Validate emails before sending.

## 3. Medium Impact Changes

### 3a. PHP Return Types on Extended Classes

If you extend any Laravel core classes and override these methods, you must add matching return types:

```php
public function count(): int {}
public function getIterator(): Traversable {}
public function getSize(): int {}
public function jsonSerialize(): array {}
public function offsetExists($key): bool {}
public function offsetGet($key): mixed {}
public function offsetSet($key, $value): void {}
public function offsetUnset($key): void {}
```

Also for `SessionHandlerInterface` implementations:
```php
public function open($savePath, $sessionName): bool {}
public function close(): bool {}
public function read($sessionId): string|false {}
public function write($sessionId, $data): bool {}
public function destroy($sessionId): bool {}
public function gc($lifetime): int {}
```

### 3b. BelongsToMany `firstOrNew` / `firstOrCreate` / `updateOrCreate`

These methods now search against the **related model's table** (not the pivot table). Review any calls like:

```php
// Before: searched pivot table
// After: searches roles table — ensure attributes match roles columns
$user->roles()->updateOrCreate(
    ['name' => 'Administrator'],
);

// firstOrCreate now accepts $values as second argument (merge on create):
$user->roles()->firstOrCreate(
    ['name' => 'Administrator'],
    ['created_by' => $user->id]  // new in 9.x
);
```

### 3c. Custom Casts & `null`

The `set` method on custom cast classes is now **always** called, even when the value is `null`. Update your casts to handle null:

```php
public function set($model, $key, $value, $attributes)
{
    if (is_null($value)) {
        return ['address_line_one' => null, 'address_line_two' => null];
    }

    if (! $value instanceof AddressModel) {
        throw new InvalidArgumentException('Expected an Address instance.');
    }

    return [
        'address_line_one' => $value->lineOne,
        'address_line_two' => $value->lineTwo,
    ];
}
```

### 3d. HTTP Client Default Timeout

The HTTP client now defaults to a **30 second timeout** (previously no timeout). Requests that previously hung indefinitely will now throw an exception.

To use a longer timeout:
```php
$response = Http::timeout(120)->get('https://example.com');
```

### 3e. Postgres `schema` → `search_path`

In `config/database.php`, rename the key for Postgres connections:

```php
// Before
'schema' => 'public',

// After
'search_path' => 'public',
```

### 3f. `assertDeleted` → `assertModelMissing`

```php
// Before
$this->assertDeleted($user);

// After
$this->assertModelMissing($user);
```

### 3g. `lang` Directory Moved

In new Laravel 9 apps, language files live in `/lang` (root) instead of `resources/lang`. If your packages publish lang files, they should use `app()->langPath()` instead of a hard-coded path. No action needed for existing apps unless restructuring.

### 3h. `password` Validation Rule → `current_password`

```php
// Before
'password' => 'password'

// After
'password' => 'current_password'
```

### 3i. `when` / `unless` Closure Behaviour

Closures passed to `when` / `unless` are now **executed** and their return value used as the boolean condition. Previously, passing any closure always evaluated to `true`.

```php
// Before: always executed (closure = truthy)
$collection->when(function ($col) {
    return false; // ignored!
}, function ($col) {
    $col->merge([1, 2, 3]); // this ran anyway
});

// After: closure return value is used as condition
$collection->when(function ($col) {
    return false; // now correctly prevents execution
}, function ($col) {
    $col->merge([1, 2, 3]); // not executed
});
```

Review all `->when()` and `->unless()` calls that pass closures as the first argument.

### 3j. Unvalidated Array Keys Now Always Excluded

The validator now automatically excludes unvalidated array keys from `validated()` data, even without specifying allowed keys in the `array` rule.

To revert to old behaviour (not recommended):
```php
// In a ServiceProvider::boot()
Validator::includeUnvalidatedArrayKeys();
```

## 4. Low Impact Changes

### Application

- **`storagePath($path = '')`** and **`langPath($path = '')`** now accept an optional `$path` argument — update custom `Application` contract implementations
- **Exception handler `ignore()`** method is now `public` instead of `protected`
- **Exception handler binding** should use `\Illuminate\Contracts\Debug\ExceptionHandler::class` type (not `\App\Exceptions\Handler::class`)

### Blade

- **`$loop` variable unavailable in `LazyCollection` loops** — accessing it would load the entire collection into memory
- **New `@checked`, `@disabled`, `@selected` directives** — may conflict with Vue events of the same name; escape with `@@checked`, etc.

### Collections

- **`reduceWithKeys` removed** → use `reduce` instead (it passes the key already)
- **`reduceMany` renamed to `reduceSpread`**
- **`Enumerable` contract** has new `sole()` method — update custom implementations

### Container

- **`Container` contract** has new `scoped()` and `scopedIf()` methods — update custom implementations
- **`ContextualBindingBuilder` contract** has new `giveConfig()` method

### Database

- **`Builder::registerCustomDoctrineType()`** removed → use `DB::registerDoctrineType()` or register in `config/database.php`

### Eloquent

- **`touch($attribute = null)`** now accepts an optional attribute argument — update overrides

### Encryption

- **`Encrypter` contract** has new `getKey()` method — update custom implementations

### Facades

- **`getFacadeAccessor()`** must return a string (container binding key), not an object instance

### Filesystem

- **`FILESYSTEM_DRIVER` env var renamed to `FILESYSTEM_DISK`** — update `.env` and `.env.example`
- **`cloud` disk** removed from default skeleton (no impact if already removed)

### Helpers

- **Global `str()` helper added** — rename any custom `str()` helper you've defined to avoid conflicts
- **`data_get()`** now works on all iterables (not just arrays and Collections)

### HTTP Client

- **Faked HTTP client now executes Guzzle middleware**
- **`Http::fake()` now affects DI-injected `Factory` instances**

### Queue

- **`opis/closure` replaced by `laravel/serializable-closure`** — remove direct usage of `opis/closure`
- **`FailedJobProviderInterface::flush($hours = null)`** — update custom implementations

### Session

- **`getSession()`** now returns `SessionInterface` or throws `SessionNotFoundException` (no longer returns `null`)

### Trusted Proxies

If importing an existing app into a fresh Laravel 9 skeleton, update `TrustProxies` middleware:

```php
// Before
use Fideloper\Proxy\TrustProxies as Middleware;
protected $headers = Request::HEADER_X_FORWARDED_ALL;

// After
use Illuminate\Http\Middleware\TrustProxies as Middleware;
protected $headers =
    Request::HEADER_X_FORWARDED_FOR |
    Request::HEADER_X_FORWARDED_HOST |
    Request::HEADER_X_FORWARDED_PORT |
    Request::HEADER_X_FORWARDED_PROTO |
    Request::HEADER_X_FORWARDED_AWS_ELB;
```

Then remove the dependency:
```bash
composer remove fideloper/proxy
```

### Validation

- **`FormRequest::validated($key = null, $default = null)`** — update overrides to accept these new arguments
