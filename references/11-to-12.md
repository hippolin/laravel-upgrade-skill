## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "laravel/framework": "^12.0"
    },
    "require-dev": {
        "phpunit/phpunit": "^11.0",
        "pestphp/pest": "^3.0"
    }
}
```

Carbon 2.x support has been **removed** — Carbon 3.x is now required. If you haven't already migrated to Carbon 3 from v11, see the [Carbon 3 migration guide](https://carbon.nesbot.com/guide/getting-started/migration.html) and review the Carbon 3 notes in the `10-to-11.md` reference.

Then run:
```bash
composer update
```

Update the global Laravel installer:
```bash
composer global update laravel/installer
```

## 2. High Impact Changes

### 2a. Dependency Updates

The key version bumps are `laravel/framework ^12.0`, `phpunit/phpunit ^11.0`, and `pestphp/pest ^3.0` (if used). No PHP version change — PHP 8.2+ is still the minimum.

### 2b. Laravel Installer

Update the global installer to support Laravel 12 and the new starter kits:

```bash
composer global update laravel/installer
```

Or, if installed via `php.new`:
```bash
# macOS
/bin/bash -c "$(curl -fsSL https://php.new/install/mac/8.4)"

# Linux
/bin/bash -c "$(curl -fsSL https://php.new/install/linux/8.4)"

# Windows (run as administrator)
Set-ExecutionPolicy Bypass -Scope Process -Force; iex ((New-Object System.Net.WebClient).DownloadString('https://php.new/install/windows/8.4'))
```

## 3. Medium Impact Changes

### 3a. `HasUuids` Trait Now Uses UUIDv7

The `HasUuids` trait now generates **UUIDv7** (time-ordered) IDs instead of UUIDv4.

If your app uses `HasUuids` and you need to keep generating **UUIDv4**, swap the trait:

```php
// Before
use Illuminate\Database\Eloquent\Concerns\HasUuids;

// After — explicitly use v4 to preserve old behavior
use Illuminate\Database\Eloquent\Concerns\HasVersion4Uuids as HasUuids;
```

If you were previously using `HasVersion7Uuids`, replace it with the standard `HasUuids` trait (which now provides the same v7 behavior):

```php
// Before
use Illuminate\Database\Eloquent\Concerns\HasVersion7Uuids;

// After
use Illuminate\Database\Eloquent\Concerns\HasUuids;
```

> ⚠️ If your app stores UUIDs externally (APIs, third-party systems, existing data) or has UUID format expectations in tests, audit all models using `HasUuids` before upgrading.

## 4. Low Impact Changes

### Concurrency Result Index Mapping

`Concurrency::run()` with an **associative array** now returns results with the original keys preserved:

```php
// Before — results returned as indexed array [0 => 2, 1 => 4]
$result = Concurrency::run([
    'task-1' => fn () => 1 + 1,
    'task-2' => fn () => 2 + 2,
]);

// After — results keyed by input key ['task-1' => 2, 'task-2' => 4]
$result = Concurrency::run([
    'task-1' => fn () => 1 + 1,
    'task-2' => fn () => 2 + 2,
]);
```

Update any code that accesses concurrency results by numeric index when using associative input arrays.

### Container Class Dependency Resolution

The DI container now **respects nullable default values** on constructor parameters instead of always resolving them:

```php
class Example
{
    public function __construct(public ?Carbon $date = null) {}
}

$example = resolve(Example::class);

// Laravel <= 11: $example->date instanceof Carbon (resolved despite null default)
// Laravel >= 12: $example->date === null (default respected)
```

If you rely on the container auto-resolving a class that has a nullable type with a `null` default, you must now either remove the default or explicitly bind it in the container.

### Image Validation Excludes SVGs

The `image` validation rule **no longer accepts SVG files** by default:

```php
// Before — SVGs were accepted
'photo' => 'required|image'

// After — SVGs rejected by default; opt-in to allow them:
'photo' => 'required|image:allow_svg'

// Or using the fluent rule:
use Illuminate\Validation\Rules\File;
'photo' => ['required', File::image(allowSvg: true)]
```

Review any file upload endpoints that use `image` validation and decide whether SVGs should be allowed.

### Local Filesystem Default Root Changed

If your app uses `Storage::disk('local')` **without** an explicit `local` disk defined in `config/filesystems.php`, the default root has changed:

- **Before:** `storage/app`
- **After:** `storage/app/private`

To restore the previous behavior, explicitly define the `local` disk in `config/filesystems.php`:

```php
'local' => [
    'driver' => 'local',
    'root'   => storage_path('app'),
    'throw'  => false,
],
```

### Multi-Schema Database Inspecting

`Schema::getTables()`, `Schema::getViews()`, and `Schema::getTypes()` now return results across **all schemas** by default (previously only the current schema):

```php
// All schemas (new default)
$tables = Schema::getTables();

// Filter to a specific schema
$tables = Schema::getTables(schema: 'main');

// Multiple schemas
$tables = Schema::getTables(schema: ['main', 'blog']);
```

`Schema::getTableListing()` now returns **schema-qualified** names by default:

```php
// Before: ['migrations', 'users']
// After:  ['main.migrations', 'main.users']
$tables = Schema::getTableListing();

// To get unqualified names:
$tables = Schema::getTableListing(schema: 'main', schemaQualified: false);
```

Update any code that iterates over `getTableListing()` output and assumes bare table names.

### Nested Array Request Merging

`$request->mergeIfMissing()` now treats **dot notation keys as nested paths**:

```php
// Before — created a literal top-level key 'user.last_name'
// After — merges into $request->input('user')['last_name']
$request->mergeIfMissing([
    'user.last_name' => 'Otwell',
]);
```

If you relied on the old behavior of creating a flat key with a dot in the name, use `merge()` with the literal key instead.

### Route Precedence Unified

When multiple routes share the same name, behavior is now consistent between cached and uncached routing: the **first** registered route wins (previously uncached routing used the last).

## 5. Very Low Impact Changes

### Authentication

- **`DatabaseTokenRepository` constructor** — `$expires` parameter now expects **seconds** instead of minutes. Only affects code that manually instantiates this class.

### Database

- **`Grammar` constructor** — now requires a `Connection` instance as its first argument. `setConnection()` method removed. Only affects custom database driver/grammar maintainers.
- **`Blueprint` constructor** — now requires a `Connection` instance as its first argument. Only affects code that manually instantiates `Blueprint`.
- **Removed/deprecated APIs:**
  - `Blueprint::getPrefix()` — deprecated
  - `Connection::withTablePrefix()` — removed
  - `Grammar::getTablePrefix()` / `setTablePrefix()` — deprecated
  - `Grammar::setConnection()` — removed
  - Use `$connection->getTablePrefix()` directly instead
