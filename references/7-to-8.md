## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "guzzlehttp/guzzle": "^7.0.1",
        "laravel/framework": "^8.0",
        "laravel/ui": "^3.0"
    },
    "require-dev": {
        "facade/ignition": "^2.3.6",
        "nunomaduro/collision": "^5.0",
        "phpunit/phpunit": "^9.0"
    }
}
```

Then run:
```bash
composer update
```

**PHP minimum version:** 7.3.0

Also upgrade your global Laravel installer to `^4.0`:
```bash
composer global require laravel/installer "^4.0"
```

## 2. High Impact Changes

### 2a. Model Factories Rewritten (Class-Based)

Laravel 8 completely rewrites model factories to be class-based. The old closure-based factories are **not compatible**.

**Option A — Use the legacy compatibility package (quickest):**
```bash
composer require laravel/legacy-factories
```
No other changes needed. Your existing `database/factories/*.php` files continue to work.

**Option B — Migrate to new class-based factories:**

Old factory (`database/factories/UserFactory.php`):
```php
use Faker\Generator as Faker;

$factory->define(App\User::class, function (Faker $faker) {
    return [
        'name' => $faker->name,
        'email' => $faker->unique()->safeEmail,
        'password' => bcrypt('password'),
    ];
});
```

New factory (`database/factories/UserFactory.php`):
```php
<?php

namespace Database\Factories;

use App\Models\User;
use Illuminate\Database\Eloquent\Factories\Factory;

class UserFactory extends Factory
{
    protected $model = User::class;

    public function definition()
    {
        return [
            'name' => $this->faker->name,
            'email' => $this->faker->unique()->safeEmail,
            'password' => bcrypt('password'),
        ];
    }
}
```

Add `HasFactory` trait to your models:
```php
use Illuminate\Database\Eloquent\Factories\HasFactory;

class User extends Model
{
    use HasFactory;
}
```

New usage in tests:
```php
// Before
$user = factory(App\User::class)->create();

// After
$user = User::factory()->create();
```

### 2b. Queue: `retryAfter` → `backoff`

Rename in all queued jobs, mailers, notifications, and listeners:

**Before:**
```php
public $retryAfter = 30;
// or
public function retryAfter()
{
    return 30;
}
```

**After:**
```php
public $backoff = 30;
// or
public function backoff()
{
    return 30;
}
```

### 2c. Queue: `timeoutAt` → `retryUntil`

**Before:**
```php
public $timeoutAt;
// or
public function timeoutAt()
{
    return now()->addMinutes(30);
}
```

**After:**
```php
public $retryUntil;
// or
public function retryUntil()
{
    return now()->addMinutes(30);
}
```

### 2d. Queue: `allOnQueue()` / `allOnConnection()` Removed (on chains)

These methods are removed when used with `withChain()`. Use `onQueue()` / `onConnection()` before `dispatch()`:

**Before:**
```php
ProcessPodcast::withChain([
    new OptimizePodcast,
    new ReleasePodcast
])->allOnConnection('redis')->allOnQueue('podcasts')->dispatch();
```

**After:**
```php
ProcessPodcast::withChain([
    new OptimizePodcast,
    new ReleasePodcast
])->onConnection('redis')->onQueue('podcasts')->dispatch();
```

> Note: `allOnQueue()` / `allOnConnection()` still work with the global `dispatch()` helper — only the `withChain()` usage is affected.

### 2e. Pagination Default: Tailwind CSS

The built-in paginator now renders Tailwind CSS classes by default (previously Bootstrap).

To keep Bootstrap styling, add to `AppServiceProvider::boot()`:

```php
use Illuminate\Pagination\Paginator;

public function boot()
{
    Paginator::useBootstrap();
}
```

### 2f. Seeder & Factory Namespaces

Seeders and factories are now namespaced.

**Step 1 — Rename directory:**
```
database/seeds/  →  database/seeders/
```

**Step 2 — Add namespace to all seeder classes:**
```php
<?php

namespace Database\Seeders; // Add this

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    public function run()
    {
        // ...
    }
}
```

**Step 3 — Update `composer.json` autoload (remove classmap, add psr-4):**

Before:
```json
"autoload": {
    "classmap": [
        "database/seeds",
        "database/factories"
    ]
}
```

After:
```json
"autoload": {
    "psr-4": {
        "App\\": "app/",
        "Database\\Factories\\": "database/factories/",
        "Database\\Seeders\\": "database/seeders/"
    }
}
```

**Step 4 — Regenerate autoloader:**
```bash
composer dump-autoload
```

## 3. Medium Impact Changes

### 3a. PHP 7.3.0 Required

Minimum PHP version raised from 7.2.5 to **7.3.0**.

### 3b. Maintenance Mode Updates (Optional)

To support pre-rendering during maintenance mode, add to `public/index.php` right after the `LARAVEL_START` definition:

```php
define('LARAVEL_START', microtime(true));

// Add these lines:
if (file_exists($maintenance = __DIR__.'/../storage/framework/maintenance.php')) {
    require $maintenance;
}
```

### 3c. `php artisan down --message` Removed

The `--message` option has been removed from `php artisan down`.

**Before:**
```bash
php artisan down --message="Upgrading Database" --retry=60
```

**After** — use `--render` to pre-render a view with your message:
```bash
php artisan down --render="errors.503" --retry=60
```

### 3d. `assertExactJson` Now Order-Sensitive for Numeric Keys

`assertExactJson` now requires numeric array keys to match in the same order.

If order doesn't matter, switch to `assertSimilarJson`:
```php
// Before (order-insensitive for numeric keys)
$response->assertExactJson(['b', 'a']);

// After — use assertSimilarJson if order doesn't matter
$response->assertSimilarJson(['a', 'b']);

// Or keep assertExactJson if order DOES matter
$response->assertExactJson(['a', 'b']);
```

## 4. Low Impact Changes

### Collections

- **`offsetExists` (i.e. `isset($collection[$key])`)** now returns `false` for `null` values (consistent with PHP's `isset`):
```php
$collection = collect([null]);
// Laravel 7: isset($collection[0]) === true
// Laravel 8: isset($collection[0]) === false
```

### Eloquent

- **`Castable::castUsing()`** now receives `array $arguments` — update custom `Castable` implementations
- **`increment()` / `decrement()`** now fire `save` and `update` model events

### Events

- **`EventServiceProvider::register()`** — must call `parent::register()` if you override this method
- **`Dispatcher` contract** — `listen($events, $listener = null)` — `$listener` is now optional

### Framework

- **`Illuminate\Support\Manager::$app`** removed → use `$container` instead
- **`elixir()` helper** removed → use [Laravel Mix](https://laravel-mix.com) instead

### Mail

- **`Mailable::sendNow()`** removed → use `send()` instead

### Routing

- **`RouteServiceProvider::$namespace`** is now `null` by default in new projects.  
  Existing apps that still have `$namespace = 'App\Http\Controllers'` are **unaffected**.  
  New preferred syntax uses PHP callable arrays:
```php
// New preferred style
Route::get('/users', [UserController::class, 'index']);

// Old string style still works if $namespace is set in RouteServiceProvider
Route::get('/users', 'UserController@index');
```

### Scheduling

- **`dragonmantank/cron-expression`** upgraded `2.x` → `3.x`. No impact unless you use it directly.

### Session

- **`Session` contract** — new `pull($key, $default)` method added — update custom implementations

### Testing

- **`TestResponse::decodeResponseJson()`** no longer accepts arguments → use `->json('key')` instead

### Validation

- **`unique` / `exists` rules** now respect model connection names (`getConnectionName()`)

### Queue (Optional)

- **Failed jobs batch support** — if using job batching, add `uuid` column to `failed_jobs`:
```php
Schema::table('failed_jobs', function (Blueprint $table) {
    $table->string('uuid')->after('id')->nullable()->unique();
});
```
Then update `queue.php`: `'failed' => ['driver' => 'database-uuids', ...]`
