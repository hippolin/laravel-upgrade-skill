## 1. Updating Dependencies

Update your `composer.json`:

```json
{
    "require": {
        "laravel/framework": "^10.0",
        "laravel/sanctum": "^3.2",
        "doctrine/dbal": "^3.0",
        "laravel/ui": "^4.0"
    },
    "require-dev": {
        "spatie/laravel-ignition": "^2.0",
        "nunomaduro/collision": "^7.0",
        "phpunit/phpunit": "^10.0"
    }
}
```

If upgrading to PHPUnit 10, also delete the `processUncoveredFiles` attribute from the `<coverage>` section of `phpunit.xml`.

Then run:
```bash
composer update
```

**PHP minimum version:** 8.1.0
**Composer minimum version:** 2.2.0

Set (or remove) minimum stability in `composer.json`:
```json
"minimum-stability": "stable"
```

## 2. High Impact Changes

### 2a. PHP 8.1.0 and Composer 2.2.0 Required

Ensure your server is running PHP ≥ 8.1.0 and Composer ≥ 2.2.0 before upgrading.

```bash
php --version
composer --version
```

Laravel 10 also makes wide use of PHP native types throughout the framework codebase. Adopting native types in your own app code is **optional** during migration but encouraged going forward.

## 3. Medium Impact Changes

### 3a. Database Expressions (`DB::raw`)

Database expressions have been rewritten. You can no longer cast a `DB::raw` expression to a string with `(string)` or `__toString()`. Instead, call `getValue()` with the grammar:

**Before:**
```php
$expression = DB::raw('select 1');
$string = (string) $expression;
```

**After:**
```php
$expression = DB::raw('select 1');
$string = $expression->getValue(DB::connection()->getQueryGrammar());
```

> This typically only affects packages or code that directly manipulates raw expressions. Standard query builder usage is unaffected.

### 3b. Eloquent `$dates` Property Removed

The long-deprecated `$dates` property on Eloquent models has been **removed**. Use `$casts` instead:

**Before:**
```php
protected $dates = ['deployed_at', 'deleted_at'];
```

**After:**
```php
protected $casts = [
    'deployed_at' => 'datetime',
    'deleted_at'  => 'datetime',
];
```

### 3c. Monolog 3.x

Laravel's Monolog dependency has been upgraded from 2.x to 3.x.

If you interact with Monolog directly (custom log handlers, processors, formatters), review the [Monolog 3 upgrade guide](https://github.com/Seldaek/monolog/blob/main/UPGRADE.md).

Also check any third-party logging packages (BugSnag, Rollbar, Sentry, etc.) for Monolog 3 + Laravel 10 compatible releases.

### 3d. Redis Cache Tags

`Cache::tags()` with the Redis driver is no longer recommended in Laravel 10. If you use Redis cache tags, consider:
- Migrating to Memcached, which fully supports tags
- Upgrading to Laravel 12.30.0+ which restores proper Redis tag support

### 3e. `MocksApplicationServices` Trait Removed

The deprecated `MocksApplicationServices` testing trait has been removed, along with the methods it provided:

| Removed method | Replacement |
|---|---|
| `expectsEvents(...)` | `Event::fake()` |
| `expectsJobs(...)` | `Bus::fake()` |
| `expectsNotifications(...)` | `Notification::fake()` |

**Before:**
```php
// In a test class using MocksApplicationServices
$this->expectsEvents(UserRegistered::class);
```

**After:**
```php
Event::fake();

// ... trigger your code ...

Event::assertDispatched(UserRegistered::class);
```

### 3f. Language Directory

In new Laravel 10 apps, the `lang` directory is no longer published by default. It can be published with:
```bash
php artisan lang:publish
```

This has **no impact on existing applications** that already have a `lang` or `resources/lang` directory.

## 4. Low Impact Changes

### Application

- **`path.public` binding removed** → use `app()->usePublicPath(__DIR__.'/public')` instead

### Authorization

- **`AuthServiceProvider::registerPolicies()`** is now called automatically by the framework. You may remove the manual call from your `boot()` method:
```php
// Can be removed:
$this->registerPolicies();
```

### Database

- **`QueryException` constructor** now takes a connection name string as its first argument — update any code that manually instantiates this exception
- **`ulid()` migration column name fixed** — previously called the column `uuid` by mistake; it is now correctly named `ulid`. If you relied on the old incorrect name, pass the name explicitly: `$table->ulid('uuid')`

### Queue

- **`Bus::dispatchNow()` and `dispatch_now()` removed** → use `Bus::dispatchSync()` and `dispatch_sync()`:
```php
// Before
Bus::dispatchNow(new MyJob());
dispatch_now(new MyJob());

// After
Bus::dispatchSync(new MyJob());
dispatch_sync(new MyJob());
```

- **`dispatch()` return value changed** — dispatching a class that doesn't implement `Illuminate\Contracts\Queue\ShouldQueue` previously returned the `handle()` result; now returns a `PendingBatch` instance. Use `dispatch_sync()` to get the handle result.

### Routing

- **`$routeMiddleware` renamed to `$middlewareAliases`** in `App\Http\Kernel` — optional rename for clarity, not required
- **`RateLimiter::attempt()` now returns the closure's return value** (previously always `true` on success):
```php
$value = RateLimiter::attempt('key', 10, fn () => ['data'], 1);
// $value is now ['data'], not true
```
- **`Redirect::home()` removed** → use a named route:
```php
// Before
return Redirect::home();

// After
return Redirect::route('home');
```

### Validation

- **Closure validation `$fail` callback** now returns an object (not void), and calling it multiple times appends messages instead of overwriting. If you type-hint the return, update accordingly:
```php
function ($attribute, $value, $fail) {
    // Can now chain:
    $fail('validation.key')->translate();
}
```

- **`$fail` with custom key** — argument order changed:
```php
// Before
$fail(['other_field' => 'Something went wrong!']);

// After
$fail('other_field', 'Something went wrong!');
```

- **Form request `after` method reserved** — if your form request has a custom `after()` method, rename it. Laravel now uses `after()` as a hook for post-validation logic.
